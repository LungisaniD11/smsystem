Step 1: User Authentication

Follow these steps to log into the system:

1 Open the application using the QR code or direct URL
2 Click "Login with Google" button on the home page
3 Select your Google account when prompted
4 Grant permissions for email and profile access
5 You will be redirected to the dashboard if you are an authorized admin
⚠️ Note: Only administrators with registered email addresses can access the system.

Step 3: Adding a New Student

To create a new student record:

1 Click "+ Add New Student" button from the dashboard
2 Fill in required fields: First Name, Last Name, Email, Mobile Number
3 Select Enrollment Status: Active or Inactive
4 Upload a profile image (optional) - JPEG or PNG format, max 5MB
5 Click "Save" to create the record
✓ Valid Image Formats: JPEG (.jpg, .jpeg) or PNG (.png) files only. The system validates file type and content to prevent invalid uploads.
6

Step 4: Searching & Viewing Students

Find and view student records:

1 Use the search bar to query by First Name, Last Name, or Student ID
2 Results are paginated (10 records per page) for easy navigation
3 Click on a student to view full details
4 Profile images appear with secure time-limited access links
🔒 Security Note: Image links expire after 15 minutes for added security. This prevents unauthorized access to student profiles.
7

Step 5: Editing Student Records

Update student information:

1 Locate the student using search or browse the enrolled students list
2 Click the "Edit" button next to the student record
3 Update any field: Names, email, phone, or enrollment status
4 Replace the profile image (optional) by uploading a new image
5 Click "Save Changes" to update the record
✏️ Tip: You can keep the existing image by leaving the image field empty during edit.
8

Step 6: Managing Student Status

Two deletion options are available:

Soft Delete (Recommended)

Mark as Inactive

  • Student record remains in system
  • Marked as inactive
  • Data is preserved
  • Can be reactivated if needed

Permanent Delete

⚠️ Permanent Deletion

  • Record is completely removed
  • Cannot be recovered
  • Use only when certain
  • Request confirmation

Step 7: Profile Image Management

Security Features

Your data is protected by:

  • Google OAuth 2.0: Industry-standard authentication
  • Admin Whitelist: Only approved users can access
  • HTTPS Encryption: All traffic is encrypted in transit
  • Anti-CSRF Tokens: Protection against cross-site attacks
  • Security Headers: HSTS, CSP, X-Frame-Options, and more
  • SAS Links: Time-limited access to stored images (15-min expiry)
  • Azure Cosmos DB: Enterprise-grade data protection
  • Azure Blob Storage: Secure media asset management

Best Practices & Tips

Recommended usage guidelines:

🔐 Security

Never share login credentials. Use only your own account for audit trails.

📋 Data Quality

Enter complete and accurate information. Verify before saving.

⏱️ Regular Backups

The system auto-backs up. Report any discrepancies immediately.

🔄 Soft Delete First

Use soft delete by default. Permanent deletion is irreversible.

✅ Verify Updates

Always confirm changes were saved successfully.

📞 Support

Contact IT support for access issues or technical problems.

Troubleshooting Guide

Common issues and solutions:

Cannot Login?

  • Verify your Google account is registered with the system admin
  • Check that your email domain is whitelisted
  • Clear browser cookies and try again
  • Use a different browser if issues persist

Image Upload Failed?

  • Ensure file is JPEG or PNG format
  • Check file size does not exceed 5 MB
  • Verify file is not corrupted

Record Not Found?

  • Try different search terms
  • Check if record was soft-deleted (marked inactive)
  • Verify student ID format
